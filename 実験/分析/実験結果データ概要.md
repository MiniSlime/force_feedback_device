# 実験結果データ概要

この文書は、`/実験/実験結果/`配下に保存されているCSVファイルの内容を確認し、各ファイルに格納されるデータ構造を整理したものです。

## フォルダ構成

- `実験前アンケート/`：参加者の基本情報
- `実験後アンケート/`：手法の好み・理由・自由記述
- `タスク記録/`：実験タスクの試行ごとの回答ログ
- `NASA-TLX/`：NASA-TLXの下位尺度と集計値

## 1. 実験前アンケート（`pre_questionnaire_*.csv`）

**ファイル名例**: `pre_questionnaire_P01_2026-01-14T02-41-55-955Z.csv`

**CSVカラム**
- `participantId`: 参加者番号（例: `P01`）
- `name`: 氏名
- `gender`: 性別（例: `男` / `女` / `無回答`）
- `handedness`: 利き手（例: `右利き` / `左利き` / `両利き` / `わからない`）

**データ特徴**
- 1行のみ（ヘッダー1行 + データ1行）
- 参加者の基本属性を記録

## 2. 実験後アンケート（`post_questionnaire_*.csv`）

**ファイル名例**: `post_questionnaire_P01_2026-01-14T02-44-00-661Z.csv`

**CSVカラム**
- `participantId`: 参加者番号
- `preferredMethod`: 好ましい手法（例: `wrist-worn` / `hand-grip` / `both` / `unknown`）
- `reason`: 理由（自由記述）
- `comments`: その他意見（自由記述）

**データ特徴**
- 1行のみ（ヘッダー1行 + データ1行）
- `reason`・`comments`に改行を含む場合があるため、CSVのクオートで囲まれて保存される

## 3. タスク記録（`experiment_*.csv`）

**ファイル名例**: `experiment_P01_wrist-worn_2026-01-14T02-26-04-620Z.csv`

**CSVカラム**
- `participantId`: 参加者番号
- `method`: 手法名（`wrist-worn` または `hand-grip`）
- `trialIndex`: 試行番号（0から開始）
- `trueDirection`: 正解方向（0, 45, 90, 135, 180, 225, 270, 315）
- `dutyCycle`: デューティ比（例: 70, 100）
- `responseAngle`: 回答角度（0-360、スキップ時は -1）
- `responseTimeMs`: 回答時間（ミリ秒）
- `error`: 誤差（度、循環誤差を考慮）
- `isCorrect`: 正答フラグ（1/0、スキップ時は -1）
- `clarity`: 力覚の分かりやすさ（1-7、スキップ時は空）
- `confidence`: 回答への確信度（1-7、スキップ時は空）

**データ特徴**
- 1ファイルに16行程度（8方向×2セット）を記録
- `method`ごとに別ファイルで保存（同一参加者でも2ファイル）

## 4. NASA-TLX（`P##_*.csv`）

**ファイル名例**: `P01_wrist_worn.csv`

**CSVカラム（列名）**
- 1列目: 項目名
- 2列目: 評価（0-100）
- 3列目: 計（ペア比較で選ばれた回数）
- 4列目: 重み（`計`/15）
- 5列目: スコア（`評価`×`重み`）

**行構成**
- 下位尺度6項目：
  - 知的・知覚的要求
  - 身体的要求
  - タイムプレッシャー
  - 作業成績
  - 努力
  - フラストレーション
- 最終行に合計スコアが保存される（項目名は空）

**データ特徴**
- 1ファイルにつき1条件（`wrist_worn` / `hand_grip`）
- 参加者ごとに2ファイル

## まとめ

実験結果のCSVは、参加者属性（実験前）、主観評価（実験後）、タスク試行ログ、NASA-TLXの4系統で整理されている。  
命名規則により参加者ID・手法・タイムスタンプが読み取れるため、参加者単位での統合や手法比較の集計が可能。
